Private Element Handling in Open DICOMweb SDK

DICOM Tag Code

A DICOM Tag Code is a 32-bit unsigned number. The 16 high-order bits are called the Group Number and the 16 low-order bits are called the Element number. Tag Codes are typically written as:

(gggg,eeee)

Where gggg is the hexadicimal Group Number and eeee is the hexadecimal Element Number.

Public and Private Tag Code

Tag Codes are divided into Public Codes and Private Code. A Public Code has a Group Number that is even number. Public Group Numbers for a Dataset are greater than 0x0007 and less than 0xFFFF.

A Private Code has a Group Number that is an odd number, which is greater than 0x0008 and less than 0xFFFE.

Private Group

A Private Group is a group of a Date Elements, where each Element has a Tag Code with the same Private Group Number.

Private Tag Code Structure

A Private Tag Code is represented as a hexadecimal integer that has the following structure:

(pppp,eeee)

Where

pppp is the private group number, which must be odd. eeee is the private element number.

The are three types of valid Private Tag codes: 1. Private Group Length, 2. Private Creator, and 3. Private Data.

There are also two potentially invalid Private Tag Codes, which might appear in the dataset:

  1. Undefined Private Tag, and
  2. Private Data without Creator.

pppp,0000

Structure of a Private Subgroup Creator Tag Code

pppp,00cc

Where

`cc` is a hexadecimal integer between `10` and `FF`.

The Private Group Tag Codes between 0001 and 000F are Undefined Private Tag Codes, which should never appear in any Dataset.

cc is the Private Creators identifier. It reserves a set a Tag codes from cc00 to ccFF, called the Private Data Subgroup for the Private Creator cc.

Structure of Valid Private Data Tag Code

pppp,ccdd

Where

`cc` is the Private Creator Index, and
`dd` is the Private Data Index between `00` and `FF`.
